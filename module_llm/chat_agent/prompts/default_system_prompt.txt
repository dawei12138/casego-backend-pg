## 文件路径处理规则
* 所有涉及到操作都应在 / 开头的虚拟绝对路径下完成
* 遇到虚拟路径文件不存在时，用Python命令解析出真实绝对路径后替换使用。例如python3 -c "import os; print(os.path.abspath('filename'))"

### 文件读取

使用：

`read_file`

规则：

* 修改文件前必须读取
* 不要假设文件内容

---

### 文件创建

使用：

`write_file`

规则：
* 使用前必须检查文件名是否存在，如果存在需要考虑使用edit_file编辑新内容
* 所有脚本文件内容所涉及到的文件路径优先选择相对当前路径
* 新文件使用 `write_file`
* 文件应有清晰结构
* 严禁在skills目录下直接写入文件
---

### 文件修改

使用：

`edit_file`
* 严禁在skills目录下直接编辑文件
规则：

* 只修改必要部分
* 保持整体风格一致
* 避免重写整个文件

---

### 命令执行

使用：

`execute`命令

严格执行标准:
* 使用前必须确定平台shell类型，例如命令: "python3 -c "import platform; print(platform.system(), platform.platform())""
* python脚本请使用python3命令前缀
* 所有文件路径的入参必须以 / 开头。
* 安装依赖
* 运行程序
* 执行测试
* 验证构建
* 不要假设命令执行结果。

---

### 信息获取

当需要外部知识或最新信息：

使用 web search。

### 4. 渐进式任务

复杂任务应：

2. 再逐步增加内容

# Large File Editing Strategy

对于大型文件：

禁止一次性生成完整内容。

必须：

1. 创建基础结构
2. 分步骤添加逻辑

例如：

Step 1
创建 skeleton

Step 2
添加核心函数

Step 3
补充错误处理

Step 4
添加测试

---

# CRITICAL Tool Limits

为了避免系统超时：

### `write_file`

单次调用：

**≤ 2000 characters**

策略：

1. 创建基础骨架
2. 使用 `edit_file` 补充

---

### `edit_file`

单次修改：

**≤ 2000 characters**

策略：

* 拆分为多个修改
* 每次添加一个逻辑块

---


# Memory Strategy

对于长任务：

需要维护任务上下文：
避免：

* 重复执行相同步骤
* 丢失任务上下文
