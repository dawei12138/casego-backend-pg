# 自定义 Faker 函数管理 - 前端开发文档

## 一、功能概述

提供可视化的 Python 代码编辑器，允许用户动态添加/修改 Faker 假数据生成函数，保存后立即生效，无需重启后端服务。

---

## 三、API 接口详情

### 基础路径

```
/api_testing/faker_config
```


---

### 3.1 获取配置内容

**请求方式**: `GET`

**请求路径**: `/api_testing/faker_config/content`

**请求参数**: 无

**响应示例**:

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "content": "#!/usr/bin/env python\n# -*- coding: UTF-8 -*-\nfrom faker import Faker\n\nfaker = Faker('zh_CN')\n\ndef faker_email():\n    \"\"\"生成随机邮箱\"\"\"\n    return faker.email()\n",
    "functions": [
      {
        "name": "faker_email",
        "doc": "生成随机邮箱",
        "params": []
      },
      {
        "name": "random_score",
        "doc": "生成随机分数",
        "params": ["min_val", "max_val"]
      }
    ]
  }
}
```

**响应字段说明**:

| 字段 | 类型 | 说明 |
|------|------|------|
| content | string | Python 源代码内容 |
| functions | array | 解析出的函数列表 |
| functions[].name | string | 函数名 |
| functions[].doc | string \| null | 函数文档字符串 |
| functions[].params | array | 参数名列表 |

---

### 3.2 校验代码语法

**请求方式**: `POST`

**请求路径**: `/api_testing/faker_config/validate`

**请求参数**:

```json
{
  "content": "def faker_email():\n    return faker.email()"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| content | string | 是 | Python 代码内容 |

**响应示例 - 校验通过**:

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "valid": true,
    "message": "语法校验通过",
    "functions": [
      {
        "name": "faker_email",
        "doc": null,
        "params": []
      }
    ]
  }
}
```

**响应示例 - 校验失败**:

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "valid": false,
    "message": "语法错误: 第 3 行 - invalid syntax",
    "functions": []
  }
}
```

**响应字段说明**:

| 字段 | 类型 | 说明 |
|------|------|------|
| valid | boolean | 语法是否正确 |
| message | string | 校验结果消息，失败时包含行号和错误信息 |
| functions | array | 校验通过时返回解析出的函数列表 |

---

### 3.3 保存配置

**请求方式**: `POST`

**请求路径**: `/api_testing/faker_config/save`

**请求参数**:

```json
{
  "content": "#!/usr/bin/env python\n# -*- coding: UTF-8 -*-\nfrom faker import Faker\n\nfaker = Faker('zh_CN')\n\ndef faker_email():\n    return faker.email()\n"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| content | string | 是 | Python 代码内容 |

**响应示例 - 成功**:

```json
{
  "code": 200,
  "msg": "保存成功",
  "data": {
    "functions": [
      {
        "name": "faker_email",
        "doc": null,
        "params": []
      }
    ]
  }
}
```

**响应示例 - 语法错误**:

```json
{
  "code": 500,
  "msg": "语法错误: 第 5 行 - invalid syntax",
  "data": null
}
```

**后端处理流程**:

1. 语法校验（AST 解析）
2. 备份原文件（.bak）
3. 写入新内容
4. 热重载模块（importlib.reload）
5. 重载失败则自动恢复备份

### 3.4 测试执行函数

**请求方式**: `POST`

**请求路径**: `/api_testing/faker_config/test`

**请求参数**:

```json
{
  "function_name": "random_score",
  "args": [60, 100]
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| function_name | string | 是 | 要测试的函数名 |
| args | array | 否 | 函数参数列表，默认 `[]` |

**响应示例 - 成功**:

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "success": true,
    "result": "85",
    "error": null
  }
}
```

**响应示例 - 函数不存在**:

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "success": false,
    "result": null,
    "error": "函数 'xxx' 不存在"
  }
}
```

**响应示例 - 执行出错**:

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "success": false,
    "result": null,
    "error": "TypeError: random_score() takes 2 positional arguments but 3 were given"
  }
}
```

**响应字段说明**:

| 字段 | 类型 | 说明 |
|------|------|------|
| success | boolean | 执行是否成功 |
| result | string \| null | 函数返回值（转为字符串） |
| error | string \| null | 错误信息 |

### 4.3 交互逻辑示例

```typescript
import { ref, onMounted } from 'vue'
import { ElMessage } from 'element-plus'
import { debounce } from 'lodash-es'

const editorContent = ref('')
const functionList = ref([])
const testResult = ref('')

// 页面加载
onMounted(async () => {
  const res = await api.get('/api_testing/faker_config/content')
  if (res.code === 200) {
    editorContent.value = res.data.content
    functionList.value = res.data.functions
  }
})

// 实时校验（防抖 500ms）
const onContentChange = debounce(async (content: string) => {
  const res = await api.post('/api_testing/faker_config/validate', { content })
  if (res.data.valid) {
    ElMessage.success('语法正确')
    functionList.value = res.data.functions
  } else {
    ElMessage.error(res.data.message)
    // 可以在编辑器中标记错误行
  }
}, 500)

// 保存配置
async function onSave() {
  const res = await api.post('/api_testing/faker_config/save', {
    content: editorContent.value
  })
  if (res.code === 200) {
    ElMessage.success('保存成功，已热重载生效')
    functionList.value = res.data.functions
  } else {
    ElMessage.error(res.msg)
  }
}

// 测试函数
async function onTestFunction(funcName: string, args: any[] = []) {
  const res = await api.post('/api_testing/faker_config/test', {
    function_name: funcName,
    args: args
  })
  if (res.data.success) {
    testResult.value = res.data.result
    ElMessage.success(`执行结果: ${res.data.result}`)
  } else {
    ElMessage.error(res.data.error)
  }
}
```

## 六、注意事项

1. **代码安全**: 编辑器中的代码会在服务端执行，请勿编写危险操作（如文件删除、网络请求等）
2. **语法要求**: 必须是合法的 Python 代码，保存前会进行 AST 语法校验
3. **函数命名**: 建议使用 `faker_` 或业务相关前缀，避免与内置函数冲突
4. **参数类型**: 模板调用时参数会通过 `ast.literal_eval` 解析，支持 int/float/str/list/dict
5. **热重载**: 保存成功后立即生效，无需重启服务

| code | 说明 |

| 500 | 操作失败（语法错误、保存失败等） |
