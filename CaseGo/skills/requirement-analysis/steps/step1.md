# Step 1: 需求收集与澄清

**标记：`[RA-STEP-1-DONE]`**

## 文件操作

- 读取：无（首步）
- 写入：`/ra_step1_input_summary.md`
- 更新：`/_progress.json`（step1 → in_progress → completed）

## 任务

仔细阅读用户提供的需求信息，识别核心意图和范围边界。

## 输出格式

写入 `/ra_step1_input_summary.md`：

```markdown
# Step 1: 需求收集与澄清

## 需求输入摘要
- 输入形式: [功能描述 / 需求文档 / 业务场景 / 混合]
- 原始描述: [保留用户原文]
- 涉及系统/模块: [识别出的系统范围]
- 信息完整度评估: [充分 / 基本充分 / 需要补充]

## 需要澄清的问题（按优先级排列）

- [P0-阻塞] 问题1: xxx（不搞清楚无法继续分析）
- [P1-重要] 问题2: xxx（影响用例覆盖度）
- [P2-建议] 问题3: xxx（有助于提高用例质量）

## 假设条件
- [跳过 P1/P2 问题时，在此列出假设]
```

## 规则

- **P0 问题必须暂停**等待用户回答后再继续，这里必须多轮调用ask_user_question，来和用户进行交互
- P1/P2 问题列出后可继续，但须标注假设条件
- 信息充分时标注"无需额外澄清"，直接进入 Step 2
