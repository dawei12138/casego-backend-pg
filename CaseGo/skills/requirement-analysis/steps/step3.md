从需求中提取所有显式和隐式的业务规则。

## 输出格式

写入 `ra_step3_business_rules.md`：

```markdown
# Step 3: 业务规则提取

| ID | 规则名称 | 规则类型 | 规则描述 | 关联功能 |

| R1 | 手机号格式校验 | 数据校验 | 手机号必须为11位数字，以1开头 | F1 |

## 隐式规则（从需求中推导，需确认）
- IR1: [推导出的隐式规则及推导依据]
- IR2: ...
```

## 规则类型

数据校验、唯一性、权限控制、状态流转、时效性、计算逻辑、安全、并发控制、容量限制、UI交互

## 注意

- 隐式规则必须明确标注为隐式，并附推导依据
- 每条规则必须关联到具体功能点
